# SKILLS	
- /kaiwu-algo-implementation：算法实现的完整流程，需要先确认当前开发状态：继续开发或是从零开发，锚点是看当前有无未完成的算法实现，如果没有就从算法调研取算法开始实现，如果有的话就从DEV_MEMORY开始继续开发。开发过程中如需要查询开发文档，可以调用相应skill。开发流程大概是：算法调研--＞取当前要实现的算法--＞初始化当前算法文档--＞实现算法，测试，commit，改 changelog 和 todo，把算法实现的细节和接口写入 算法文档，把经验写入项目经验--＞整合当前算法 文档 成为这个算法详细 文档，并且更新算法总表--＞整合项目经验，changelog，todo，把有意义的放到全局--＞最终 commit，push 远程，当前算法完整实现。
- /kaiwu-doc-query：按照规则从开发文档中查询信息，并且维护EXPERIENCE.md（目前README中没写这个需要加上）。
- /kaiwu-current-algo-analysis：分析当前算法实现：先从算法总表了解当前算法基本信息，然后定位到其对应的详细文档分析。
- /kaiwu-dev-init：拉取最新代码，创建开发分支，阅读README文档和WORKFLOW。
- /kaiwu-version-control：版本控制，当算法实现阶段性完成时提交并且写明干了什么，修改了什么，当算法完全实现时的提交需要注明实现的算法的名字。
- /kaiwu-memory-archive：执行DEV_MEMORY->GLOBAL_DOCS的归档，把开发文档变成全局文档中该算法的详细文档，其他同名文件整理，在过滤掉无用内容后把可复用的内容添加到全局中。在结束后需要将DEV_MEMORY重置。
- /kaiwu-full-iteration：执行从初始化、算法实现到信息归档、版本管理的全流程。
# CONTRIBUTE.md
包含分支规范
	`main`分支是主分支，`feature/*`是开发分支
开发规范
	先git pull拉取最新代码——》创建开发分支——》开发——》小步改动测试+commit——》必须完整实现+测试通过+训练完成+信息归档——》才能合并到主分支
提交规范
	对于小步改动需要注明做了什么，改了哪些文件，接口。完整改动需要注明算法名。

# 面向人类开发者的开发文档.md
人类开发者主要是使用codex等ai开发工具进行开发，该文档需要告诉人进行该项目开发的前置知识和准备，需要怎么进行人机协作，git协作，多人协作。
- 前置知识和准备：
	- 理解项目的定义和任务。
	- 熟悉README和workflow。
	- 熟悉开发文档。
	- 熟悉代码文件、超参设计和训练日志。
	- 熟悉开发流程及git协作。
- 人机协作，git协作在其他文件有，直接复制过来。
- 多人协作：
	- 每次开发必须拉取最新代码，每次合并必须是完整算法实现+测试跑通+训练结束+DEV_MEMORY归档到GLOBAL_DOCS+完整版本管理才行。
	- 如有冲突自行判断，或在群里及时沟通。
