# 开发文档索引与 AI 查询指南

本文件是面向 AI 助手使用的配套索引文档，用于帮助 AI 根据用户需求，快速定位到正确的开发文档位置，并以更稳定的方式组织回答。

> 如果你只想先了解全貌，请优先阅读 [开发文档.md](开发文档.md)。  
> 如果你已经知道问题类型，请直接根据下面的“按需求导航”或“关键词索引”跳转。

---

## 1. 这份文档的用途

这份文档不重复展开所有细节，而是承担三个作用：

1. 索引入口：告诉 AI 某类问题应该优先查哪份文档。
2. 查询导航：把常见需求映射到具体文件和章节类型。
3. 回答约束：提醒 AI 在回答时区分环境规则、训练奖励、框架接口、数据协议，避免混淆。

---

## 5. AI 查询规则

### 5.1 先做问题分类

先判断问题属于哪一类：

- 赛题规则
- 环境配置
- 环境返回数据 / 数据协议
- 智能体实现
- 算法 / 模型 / 工作流
- 监控与日志

### 5.2 再按层级查文档

推荐顺序：

1. 先查 [开发文档.md](开发文档.md) 获取整体定位。
2. 再查拆分文档获取精确信息。
3. 如果是字段、类型、枚举值问题，最后一定回到 [开发指南/数据协议.md](开发指南/数据协议.md) 做确认。

### 5.3 回答参数问题时的输出模板

如果问题是“某个配置项/字段是什么意思”，建议回答时尽量包含：

- 字段名
- 含义
- 数据类型
- 取值范围
- 默认值
- 影响 / 使用场景

### 5.4 回答接口问题时的输出模板

如果问题是“某个函数怎么用”，建议回答时尽量包含：

- 函数名
- 输入参数
- 返回值
- 调用时机
- 相关代码位置 / 文档位置

### 5.5 文档未明确说明时不要强行推断

如果文档没有明确给出结论，建议明确说明：

- 文档未明确说明该细节
- 以下为基于现有描述的合理推断，不应视为官方协议

## 6. 常见混淆点

| 容易混淆的概念 | 正确理解 |
| --- | --- |
| Score vs Reward | `Score` 是环境任务表现分数；`Reward` 是强化学习训练奖励，两者用途不同，不能混用 |
| 16 维动作 vs 8 维基线动作 | 环境原生动作空间是 16 维（8 移动 + 8 闪现），但当前基线算法默认只使用 8 维移动动作 |
| 训练配置 vs 评估配置 | 训练通常改 `train_env_conf.toml`；评估通常在平台界面中配置 |
| `observation` vs `extra_info` | `observation` 偏局部观测；`extra_info` 提供额外全局信息和结果信息 |
| `terminated` vs `truncated` | `terminated` 表示正常终止条件触发；`truncated` 表示达到最大步数或异常中断 |
| 环境文档 vs 框架文档 | 环境文档回答“这个赛题是什么、怎么运行”；框架文档回答“接口怎么接、训练流程怎么实现” |

## 7. 推荐回答方式

为了让 AI 输出更可信，建议在回答时带上“来源文档 + 主题”，例如：

- 根据《开发指南/环境详述》的环境配置部分，`monster_interval` 表示第二个怪物出现间隔……
- 根据《开发指南/数据协议》的 `EnvInfo` 协议，`flash_count` 表示闪现技能已使用次数……
- 根据《腾讯开悟强化学习框架/智能体/工作流开发》的 `workflow` 示例，训练流程包括环境交互、样本处理和模型更新……

如果用户要的是直接结论，可以先给简答，再附依据来源。
