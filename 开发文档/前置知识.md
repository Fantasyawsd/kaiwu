## 当前监控面板设计

当前监控设计以代码实现为准，配置入口在 `agent_ppo/conf/monitor_builder.py`。

### 指标来源

- `agent_ppo/workflow/train_workflow.py`
  - 负责训练单局指标上报。
  - 负责验证阶段聚合指标上报。
- `agent_ppo/algorithm/algorithm.py`
  - 负责 PPO loss、优势统计、return 统计上报。
- 环境/平台原生指标
  - 包括地图、步数、宝箱、闪现、buff、怪物等面板。
  - `monitor_builder.py` 只做分组、面板命名和 Prometheus 表达式拼装。

### 分组布局

当前监控面板共 4 组：

| 分组名 | `group_name_en` | 含义 | 实际来源 |
|------|------------------|------|---------|
| `Train` | `algorithm` | 训练单局表现 | `train_workflow.py` |
| `环境指标` | `env` | 环境配置与环境状态 | 环境 / 平台原生指标 |
| `训练损失` | `algorithm` | PPO 优化过程统计 | `algorithm.py` |
| `Val` | `algorithm` | 验证阶段均值指标 | `train_workflow.py` |

这里要注意两件事：
- `group_name_en` 不只是注释字段，而是平台归类的关键值，当前实现使用 `algorithm` 和 `env`。
- `name` 主要影响展示标题；`name_en` / `metrics_name` 对应稳定面板标识与真实指标键，改动时要一起核对。

### Train 组

Train 组按训练单局上报，当前包含以下面板：

- 基础训练指标
  - `train_reward`
  - `train_total_score`
  - `train_step_score`
  - `train_treasure_score`
  - `train_treasures_collected`
  - `train_episode_steps`
- 过程细分指标
  - `train_speedup_reached`
  - `train_pre_speedup_steps`
  - `train_post_speedup_steps`
- reward 拆分
  - `train_pre_speedup_reward`
  - `train_post_speedup_reward`
  - `train_pre_speedup_shaped_reward`
  - `train_post_speedup_shaped_reward`
- gain 拆分
  - `train_pre_speedup_step_score_gain`
  - `train_post_speedup_step_score_gain`
  - `train_pre_speedup_treasure_gain`
  - `train_post_speedup_treasure_gain`
  - `train_pre_speedup_total_score_gain`

当前实现说明：
- `train_pre_speedup_reward = pre_speedup_shaped_reward + pre_speedup_terminal_bonus`
- `train_post_speedup_reward = post_speedup_shaped_reward + post_speedup_terminal_bonus`
- 当前 Train 组没有 `train_post_speedup_total_score_gain`

### 环境指标组

环境指标组由 `monitor_builder.py` 组织展示，当前面板如下：

- `得分`
  - `total_score`
  - `treasure_score`
  - `step_score`
- `地图`
  - `total_map`
  - `map_random`
- `步数`
  - `max_steps`
  - `finished_steps`
- `宝箱`
  - `total_treasure`
  - `treasures_collected`
- `闪现`
  - `flash_count`
  - `flash_cooldown`
- `加速增益`
  - `total_buff`
  - `collected_buff`
  - `buff_refresh_time`
- `怪物移动速度`
  - `monster_speed`
- `怪物出现间隔`
  - `monster_interval`

实际实现里，`max_steps` 这个展示项使用的表达式是 `avg(max_step{})`，也就是展示名和环境原始键名并不完全一致。

### 训练损失组

训练损失组由 `Algorithm.learn()` 上报，当前包含：

- `reward`
- `total_loss`
- `value_loss`
- `policy_loss`
- `entropy_loss`
- `grad_clip_norm`
- `clip_frac`
- `explained_var`
- `adv_mean`
- `ret_mean`

其中：
- `adv_mean` 对应原始 advantage 的 batch 均值。
- `ret_mean` 对应 `reward_sum` 的 batch 均值。
- 这一组当前按约 60 秒节流一次，不是每个 batch 都推到 monitor。

### Val 组

Val 组不是逐局即时上报，而是 eval 阶段聚合后统一上报。

当前实现节奏：
- 每训练 50 局，切换一次 eval。
- eval 默认跑 10 局。
- 10 局结束后，对各验证指标取均值并上报到 Val 组。

当前包含以下指标：

- 基础验证指标
  - `val_reward`
  - `val_total_score`
  - `val_step_score`
  - `val_treasure_score`
  - `val_treasures_collected`
  - `val_episode_steps`
- 过程细分指标
  - `val_speedup_reached`
  - `val_pre_speedup_steps`
  - `val_post_speedup_steps`
- reward 拆分
  - `val_pre_speedup_reward`
  - `val_post_speedup_reward`
  - `val_pre_speedup_shaped_reward`
  - `val_post_speedup_shaped_reward`
- gain 拆分
  - `val_pre_speedup_step_score_gain`
  - `val_post_speedup_step_score_gain`
  - `val_pre_speedup_treasure_gain`
  - `val_post_speedup_treasure_gain`
  - `val_pre_speedup_total_score_gain`
  - `val_post_speedup_total_score_gain`
- 终止相关
  - `val_pre_speedup_terminal_bonus`
  - `val_post_speedup_terminal_bonus`
  - `val_post_speedup_terminated`
  - `val_terminated_rate`
  - `val_completed_rate`
  - `val_abnormal_truncated_rate`
- 结尾状态
  - `val_danger_level`
  - `val_nearest_treasure_dist`

### 监控面板排查要点

如果平台里面板没有出现，优先检查下面几项：

- `group_name_en` 是否仍然是平台可识别的分类值。
- `name_en` 是否保持稳定且没有和其他面板冲突。
- `metrics_name` / `expr` 是否对应真实会上报的指标键。
- 训练代码里是否真的调用了 `monitor.put_data(...)`。
- Val 组是否已经达到 eval 触发条件；如果还没跑满评估轮次，Val 不会立刻出现新曲线。

## 一、赛题与环境规则

### 1.1 任务目标

**路径规划 - 生存博弈**

控制鲁班七号在 1000 步（可配置）内：
- 躲避 2 个怪物的追击
- 尽可能收集宝箱提高得分

### 1.2 地图与元素

| 元素 | 说明 |
|------|------|
| **地图** | 128×128 栅格地图，坐标原点(0,0)在左上角，x向右、z向下 |
| **英雄** | 鲁班七号，视野范围 21×21（各方向10格），默认移速1格/步 |
| **怪物** | 2个，怪物1开局出现，怪物2在第300步（可配置）出现 |
| **宝箱** | 最多10个，每个100分，随机分布在道路上 |
| **加速buff** | 最多2个，拾取后移速翻倍（2格/步），持续50步 |
| **障碍物** | 阻碍移动，遇障碍会停在最后可通行格 |

### 1.3 动作空间（16维）

| 动作值 | 类型 | 方向 | 说明 |
|--------|------|------|------|
| 0-7 | 移动 | 8方向 | 每步移动1格（有buff时2格） |
| 8-15 | 闪现 | 8方向 | 直线10格/斜向8格，冷却100步 |



### 1.4 计分规则

```
任务得分 = 步数得分 + 宝箱得分
步数得分 = 完成步数 × 1.5
宝箱得分 = 收集宝箱数 × 100
```

### 1.5 关键配置参数

环境配置：`agent_算法名/conf/train_env_conf.toml`

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `map` | 训练地图列表 [1-10] | [1,2,...,10] |
| `map_random` | 是否随机选图 | false |
| `treasure_count` | 宝箱数量 0-10 | 10 |
| `buff_count` | 加速buff数量 0-2 | 2 |
| `monster_interval` | 第二怪物出现步数 | 300 |
| `monster_speedup` | 怪物加速步数 | 500 |
| `max_step` | 单局最大步数 | 1000 |
| `talent_cooldown` | 闪现冷却步数 | 100 |

## 二、数据协议

### 2.1 核心数据结构

#### observation（观测）

| 字段 | 类型 | 说明 |
|------|------|------|
| `step_no` | int32 | 当前步数 |

#### FrameState

| 字段 | 类型 | 说明 |

| 字段 | 说明 |

## 三、框架接口

### 3.1 环境接口

```python
# 重置环境
obs, state = env.reset(usr_conf=usr_conf)

# 执行动作
frame_no, _obs, score, terminated, truncated, _state = env.step(act)
```

- `terminated`: 被怪物捕获，任务失败
- `truncated`: 达到最大步数或异常中断

### 3.2 Agent 核心接口

```python
# 观测处理：原始观测 → 特征向量
def observation_process(self, obs, state) -> ObsData:
    return ObsData(feature=feature, legal_act=legal_actions)

# 动作处理：预测输出 → 环境动作
def action_process(self, act_data):
    return act_data.act

# 预测（训练时，采样）
def predict(self, list_obs_data, list_state) -> [ActData]

# 贪婪预测（评估时）
def exploit(self, observation) -> action

# 训练
def learn(self, list_sample_data)

# 模型加载/保存
def load_model(self, path=None, id="1")
def save_model(self, path=None, id="1")
```

### 3.3 特征处理接口

```python
# 奖励计算
def reward_shaping(obs, _obs, state, _state) -> reward

# 样本处理（轨迹 → 训练样本）
def sample_process(self, list_game_data) -> [SampleData]

# 分布式传输转换（需 @attached 装饰器）
def SampleData2NumpyData(g_data) -> np.array
def NumpyData2SampleData(s_data) -> SampleData
```

## 五、实现参考

### 5.1 特征工程（40维）

| 部分 | 维度 | 内容 |
|------|------|------|
| hero_self | 4 | 位置x/z归一化、闪现冷却、加速buff剩余 |
| monster_1 | 5 | 是否在视野、位置x/z、速度、距离 |
| monster_2 | 5 | 同上（未生成时全0） |
| map_local | 16 | 4×4局部地图通行性 |
| legal_action | 8 | 移动方向合法掩码 |
| progress | 2 | 步数归一化、存活比例 |

### 5.2 网络结构

```
输入(40D) → MLP(40→128→64) → Actor头(64→8)  → 动作logits
                          → Critic头(64→1) → 状态价值
```

### 5.3 奖励设计

| 奖励项 | 数值 | 说明 |
|--------|------|------|
| survive_reward | +0.01/步 | 基础生存奖励 |
| dist_shaping | ±0.1×Δdist | 远离怪物正奖励 |

### 5.4 PPO 损失函数

```
total_loss = vf_coef × value_loss + policy_loss - beta × entropy_loss
```

## 六、常见混淆点

| 概念                            | 说明                                  |
| ----------------------------- | ----------------------------------- |
| **Score vs Reward**           | Score是环境任务得分（平台评分）；Reward是强化学习训练奖励  |
| **16维 vs 8维**                 | 环境原生16维（移动+闪现）                      |
| **训练配置 vs 评估配置**              | 训练改 `train_env_conf.toml`；评估在平台界面配置 |
| **observation vs extra_info** | observation是局部视野；extra_info含全局信息    |
| **terminated vs truncated**   | terminated=被抓住；truncated=到最大步数或异常   |
| **map_info vs 局部地图**          | map_info是21×21完整视野；                 |
