# 算法总表

本文件是当前仓库已实现算法的统一清单，记录算法状态、训练得分与文档路径。

职责边界：

- 本文件由 AI 维护，用于反映仓库内已经实现并落库的算法状态。
- 本文件只记录已实现算法；尚未实现的候选方向不写在这里。
- 未实现的外部调研候选方案统一记录在 `GLOBAL_DOCS/算法调研.md`。
- 当前正在实现但尚未完成的工作统一记录在 `DEV_MEMORY/*`。

- 若某个算法被确定为仓库新的默认开发起点（通常即当前稳定主线；若已有正式训练结果可一并记录），必须在本文件中切换“当前基线”标记。
- `kaiwu-memory-archive` 在归档当前稳定主线版本时，必须同步维护“当前基线”角色，只允许保留一个“当前基线”条目；旧基线需降级为历史基线或已归档版本。
- **当前算法实现源码目录固定为 `agent_ppo/`（目录名源于最初 PPO 实现，但不限制算法类型，可以是 DQN、SAC 等任意 RL 算法）；`agent_diy/` 仅作为模板，不作为正式算法实现目录。**
- 若需定位某个历史算法实现，优先通过“完整算法 commit 中标注的算法完整名 + git 回退”定位，而不是依赖源码地址字段。

## 字段说明

| 字段    | 说明                                                                |
| ----- | ----------------------------------------------------------------- |
| 算法名   | 算法标识（对应 `conf/algo_conf_gorge_chase.toml` 中的 key）                 |
| 算法完整名 | 当前这一次完整算法实现的唯一名字，建议与正式算法文档目录名保持一致；完整算法 commit 必须显式包含该名字 |
| 算法描述  | 简要说明算法思路与特点                                                       |
| 训练得分  | 可确认的平台训练得分；无正式结果时写“暂无正式训练得分”                                      |
| 算法文档  | 正式算法文档入口路径（使用 `GLOBAL_DOCS/算法文档/<算法完整名>/README.md`）                     |
| 备注    | 当前状态说明、已知限制、下一步                                                   |
