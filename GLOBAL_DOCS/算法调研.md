# 算法调研

本文件用于记录外部调研得到、但尚未在仓库内实现的候选算法与优化方向。

## 职责边界

- 本文件主要由人维护。
- 只记录未实现方向，不记录已经落库的正式算法。
- 一旦某个方向被实现并稳定落库，应从这里迁移到 `GLOBAL_DOCS/算法总表.md` 与正式算法文档。

## 源码存放位置

外部调研得到的算法完整源码存放在 **`reference_algos/`** 目录下：

```
reference_algos/
├── <算法名>/              # 每个外部算法一个子目录
│   ├── README.md          # 该算法的基本说明、来源、核心思路
│   ├── code/              # 源码实现
│   └── docs/              # 相关文档、笔记
```

**使用规则**：

- `reference_algos/` 中的代码**仅供参考借鉴**，不作为正式训练入口
- 设计新算法时，应优先查阅 `reference_algos/` 中的源码，提取可迁移的组件
- 实现算法时，应将可复用的思路迁移到 `agent_ppo/` 中，而非直接修改 `reference_algos/`
- 每个 `reference_algos/<算法名>/` 应包含 README.md 说明该算法的来源、核心思想和可借鉴点

## 建议字段

- 方向名
- 来源（论文 / 博客 / 讨论 / `reference_algos/` 中的具体目录）
- 核心想法
- 可参考的源码路径（如 `reference_algos/hok_prelim/code/agent.py`）
- 为什么适合 Gorge Chase
- 预期改动文件
- 风险
- 优先级
- 当前状态
