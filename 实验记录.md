# 实验记录

用于记录本仓库里已经尝试过的方法、配置改动与对应指标，避免重复试验，也方便后续横向比较。

## 记录原则

- 每次只记录一个**可复现的尝试**。
- 先写清楚**改了什么**，再写**指标结果**。
- 指标要尽量注明来源：训练日志、monitor、平台评估结果。
- 如果一次实验改了多个因素，请在“变量控制”里明确说明。

## 建议重点记录的指标

结合当前代码，优先记录这些指标：

- 训练过程指标
  - `total_loss`
  - `policy_loss`
  - `value_loss`
  - `entropy_loss`
  - `reward`
- 对局/episode 指标
  - `episode_steps`
  - `episode_cnt`
  - `total_reward`
  - `result`（`WIN` / `FAIL`）
- 环境表现指标
  - `sim_score` / `total_score`
- 补充信息
  - 是否使用 `ppo` / `diy`
  - 是否修改了 `train_env_conf.toml`
  - 是否修改了 `conf/configure_app.toml`
  - 是否扩展了动作空间、奖励函数、特征工程、模型结构
